Summary

For this lab, I will be deploying and showing off the Materialize Framework, aimed at creating a unified user experience with CSS and JS.

Setup

Materialize was very simple to setup, only requiring two CDN lines listed below:

Required Imports

<script src="https://cdnjs.cloudflare.com/ajax/libs/materialize/1.0.0/js/materialize.min.js"></script>

<link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/materialize/1.0.0/css/materialize.min.css">

The benefit to using materialize, is that it allows me to quickly create aesthetically pleasing, responsive and standardized CSS styles, all in a few class definitions; making materialize very fast to get started.
I found that I could quickly build the current layout with little to no issues after navigating the documentation provided by materialize.
Materialize lets me use fluid grid based classification, much like bootstrap but having the benefit of being more readable, in my opinion.
I also found appreciation in the provided colour chart from materialize, allowing me to pick complimenting colours quickly and efficiently.

Issues

Some issues I initially encountered, is with the design of the language being very <div> heavy. It is very easy to get lost in this HTML code when working in Materialize.
I also encountered some wrapping issues when trying to add text to certain classes. They were swiftly resolved due to easily accessible documentation. There were also issues in regards to javascript function loading. M.AutoInit(); was a mandatory js function not immedietely apparent - which caused some confusion.

On the sidebar, I have listed some of Materialize's featuers - which are highly focused on stylizing with both CSS and JS.
